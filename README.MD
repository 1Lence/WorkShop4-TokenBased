Приложение полностью соответствует тз, token-based реализован с помощью JWT.
Имеется 3 контроллера, каждый настроен под свою роль.
В админ контроллере можно изменить любые данные существующего пользователя.
Зарегистрировать нового пользователя под существующим логином или почтой нельзя.
Реализовал удобную систему ролей, а конкретнее у каждой роли имеется своё разрешение, по которому идет проверка.

URLS:
- POST: http://localhost:8080/api/auth/registration регистрация
пример валидный JSON
{
    "login":"Kirill",
    "password": "1234567",
    "email": "email@gmail.com",
    "role": "ADMIN"
}
- POST: http://localhost:8080/api/auth/sign-in Логин, выдающий токен
JSON:
{
   "login":"Kirill",
   "password":"1234567"
}
- POST http://localhost:8080/api/auth/refresh Обновить токен
{
    "refreshToken":"Тут ваш токен"
}
- POST http://localhost:8080/api/logout - отозвать токен
